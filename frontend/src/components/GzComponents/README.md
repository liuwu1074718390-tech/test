## 广咨组件库

### 1.组件库介绍

#### 1.1 组件库概述

组件库是前端开发中，将常用的功能模块封装成可复用的组件，以实现代码的复用和减少重复开发的工作量。组件库通常包括按钮、表单、表格、弹窗等常用组件，以及一些高级组件如图表、地图等。

#### 1.2 组件库的优势

1. 提高开发效率：组件库可以减少重复开发的工作量，提高开发效率。
2. 统一风格：组件库可以统一项目的UI风格，提高用户体验。3. 降低维护成本：组件库可以降低项目的维护成本，因为组件库中的组件通常经过严格测试，可以保证稳定性和兼容性。

#### 1.3 组件库的组成

组件库通常包括以下几部分：

1. 基础组件：包括布局、表单、表格、弹窗等常用组件，不涉及业务逻辑，不包含接口，只提供样式和功能。
2. 业务组件：包括业务相关的组件，如选择用户，选择组织，上传文件等等业务，通常需要与后端接口进行交互，实现业务逻辑。需要根据当前业务逻辑来调整接口与配置逻辑。

#### 1.4 常见问题

1. 组件样式错乱：组件样式可能与项目使用的样式冲突，需要根据项目使用的样式进行适配和补全。
2. 组件功能不完善：组件功能可能不满足业务需求，需要根据业务需求进行调整和扩展。
3. 业务组件依赖问题：业务组件可能依赖于其他业务组件，需要根据业务需求进行依赖关系调整。
4. 业务组件接口错误：业务组件的接口可能与后端接口不一致，需要根据后端接口进行调整。

#### 1.5 组件库的更新与维护

组件库的更新和维护是持续的过程，需要根据业务需求和技术发展进行迭代和优化。以下是一些常见的更新和维护方法：

1. 因为组件库还在建设中，为了组件更好的拓展使用，需要根据业务需求进行相应的调整和优化。所以未对组件库进行npm包化，后续有新增组件，目前也是在此项目中更新
2. 持续更新：根据业务需求和技术发展，持续更新组件库，增加新的组件和功能，修复已知问题。
3. 升级维护：升级维护是指对组件库进行升级和升级维护，包括升级组件库的版本、升级组件库的依赖库、升级组件库的样式和功能等。升级维护的目的是保持组件库的兼容性、安全性和性能。
4. 后续会根据团队及项目使用情况，将组件库进行npm包化，方便团队内部使用。
